## Blocker

### B-1. openapi 定義 `/api/v2/tags:cursor` 但無實作註冊，執行期會 500
- 檔案：`doc/openapi.yml:42-55`（定義 `GetTagsCursor`，tag=`Tag`）
- 檔案：`src/be/JobBank1111.Job.WebAPI/Contract/AutoGenerated/ContractControllers.cs:63-100`（`TagController` 建構子需注入 `ITagController _implementation`）
- 檔案：`src/be/JobBank1111.Job.WebAPI/Program.cs:55`（僅註冊 `IMemberController -> MemberV2ControllerImpl`，**無** `ITagController` 註冊）
- 問題：ASP.NET Core 會掃描到 `TagController`，路由 `api/v2/tags:cursor` 存在，但請求進來時 DI 無法解析 `ITagController` → 拋 `InvalidOperationException`（HTTP 500）。openapi 有此端點卻無對應實作。
- 嚴重度：blocker

## Major

### M-1. openapi 只有 v2，實作卻有 v1 members（規格 vs 實作漂移）
- 檔案：`doc/openapi.yml:42,109,136,154`（全部為 `/api/v2/...`）
- 檔案：`src/be/JobBank1111.Job.WebAPI/Member/MemberController.cs:11,29,63`（`MemberV1Controller`，路由 `api/v1/members:cursor`、`api/v1/members:offset`、`api/v1/members` POST）
- 問題：v1 members 三個端點已實作並啟用，但 openapi.yml 完全未定義 v1；反向地，openapi 的 v2 tags 端點無實作（見 B-1）。規格與實作雙向不一致，Client SDK（Refitter 產生）不會包含 v1。
- 嚴重度：major

### M-2. 根 `CLAUDE.md` 所有檔案連結指向不存在的 `project-template/`
- 檔案：`/mnt/d/lab/api.template/CLAUDE.md`（全篇 📝 連結，如 `project-template/src/be/JobBank1111.Job.WebAPI/Member/MemberController.cs`）
- 問題：實際目錄為 `dotnet-project-template/`，但 CLAUDE.md 所有參考連結前綴為 `project-template/`，目錄不存在 → 所有連結失效。被連結的檔案本身存在（於 `dotnet-project-template/src/be/...`），純屬路徑前綴錯誤。
- 嚴重度：major

### M-3. `CLAUDE.md` 宣稱的多個 task 在 Taskfile.yml 不存在
- 檔案：`/mnt/d/lab/api.template/CLAUDE.md`（「常用指令」段）對照 `dotnet-project-template/Taskfile.yml`
- 缺少的 task：`build`、`test-unit`、`test-integration`、`ef-migration-add`、`ef-database-update`（Taskfile 實際只有 dev-init/redis-*/ef-codegen*/codegen-api*/api-dev）
- 問題：文件承諾「執行單元測試 `task test-unit`」「建立 Migration `task ef-migration-add`」等，但這些 task 都不存在，使用者照做會失敗。
- 嚴重度：major

### M-4. `env/.template-config.json` 不存在，觸發專案狀態誤判
- 檔案：`dotnet-project-template/env/`（僅 `local.env`，無 `.template-config.json`）
- 問題：CLAUDE.md「專案狀態檢測機制」以此檔存在與否判斷是否為空白專案；缺此檔會被判為「空白專案」而觸發初始化對話，但此範本其實已完整（有 .sln、src/、docker-compose.yml）→ 誤判。
- 嚴重度：major

### M-5. 根層 best-practices 文件大量重複
- 檔案：`dotnet-project-template/best-practices.md`（1786 行，「ASP.NET Core 開發指南」）
- 檔案：`dotnet-project-template/最佳實踐.md`（1311 行，「ASP.NET Core 最佳實踐指南」）
- 問題：兩份皆為同一專案的 ASP.NET Core 最佳實踐（Result Pattern、Repository、中介軟體等），內容高度重疊，再加上根 `CLAUDE.md` 與 `CLAUDE_OLD.md`（128K），同一套規範散落 4 個檔案，維護時易不同步。
