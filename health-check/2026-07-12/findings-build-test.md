## 2. 測試
### 2.1 單元測試 JobBank1111.Job.Test（xUnit）— 8/10（P2）
失敗 2：`CacheProviderFactoryTest.寫讀快取資料_String_Redis / _Json_Redis`，
原因 `RedisConnectionException: UnableToConnect on localhost:6379`——測試直連本機 Redis，非 Testcontainers 管理。
【推論】「單元測試」不隔離，依賴常駐服務；與專案測試策略（真依賴走 Testcontainers）衝突。CI 無常駐 Redis 即紅。

### 2.2 整合測試 JobBank1111.Job.IntegrationTest（MSTest+Reqnroll+Testcontainers）— 3/8（**P0**）
Testcontainers 啟停正常（非 Docker 問題）。失敗 5 場景：新增一筆會員、API Header-1、API Header-2、查詢會員 cursor、查詢會員 offset。
根因（五者堆疊一致）：
```
System.InvalidOperationException: The PipeWriter 'ResponseBodyPipeWriter' does not implement PipeWriter.UnflushedBytes.
   at System.Text.Json...SerializeAsync(PipeWriter, ...)
   at Microsoft.AspNetCore.Mvc.Formatters.SystemTextJsonOutputFormatter.WriteResponseBodyAsync(...)
```
效果：TestServer 下所有回傳 JSON 的端點在序列化回應時擲例外 → ExceptionHandlingMiddleware 轉 500 → 斷言失敗。
【⚠️ 矛盾】與封存計畫 `66e062c`「100% - all tests passing」及 2026-07-11 obs 770/773/778「整合測試通過」不符。本次健檢**證實 PipeWriter 問題目前在本機可重現、未修復**。

【根因確診】（主對話 2026-07-12 01:20 補驗，高信心）
- `JobBank1111.Job.IntegrationTest.csproj:17`：`Microsoft.AspNetCore.Mvc.Testing` **8.0.10**，而應用為 net10.0（SDK 10.0.102，無 global.json）。
- .NET 10 的 STJ `SerializeAsync(PipeWriter)` 要求 `PipeWriter.UnflushedBytes`；8.x TestServer 的 `ResponseBodyPipeWriter` 未實作 → **只有 TestServer 環境炸，Kestrel runtime 不受影響**。這解釋當初為何被誤判為「業務邏輯 HTTP 500」。
- 修法（一行，待修復輪執行）：Mvc.Testing 升至 10.0.x（併入套件統一升級）。修復效果本輪未驗證（診斷輪不改碼）。

【矛盾釐清（推論，無法回溯證實）】昨晚「8/8 通過」的執行時間僅 4–5 秒，不可能含 Testcontainers 啟停（本次實跑 26.7s）；且 obs 770 自述當時 `--no-build` 參數反覆出錯。推論昨晚驗證跑到含 SynchronousJsonOutputFormatter workaround 的舊二進位。流程教訓：完成宣告必須附 fresh build 證據與執行時間合理性檢查。
